Node.js Debugger

Node.js Debugger


>node debug app.js

  • integriert in node
  • interaktive Shell

Stepping

  • cont, c - Continue execution
  • next, n - Step next: bleibt im selben Scope (springt in keine Funktion rein)
  • step, s - Step in: springt in jede Funktion rein
  • out, o - Step out: springt aus der aktuellen Funktion raus (nächstes return oder throw)
  • pause - Pause running code (like pause button in Developer Tools)

Breakpoints

  • setBreakpoint
  • clearBreakpoint

Info

  • backtrace, bt - Print backtrace of current execution frame
  • list(5) - List scripts source code with 5 line context (5 lines before and after)
  • watch(expr) - Add expression to watch list
  • unwatch(expr) - Remove expression from watch list
  • watchers - List all watchers and their values (automatically listed on each breakpoint)
  • repl - Open debugger's repl for evaluation in debugging script's context

Client-side Bundlers

Bundler. It is better for you to not know what it is. 

// a.js
var b = require("./b");
b.stuff("It works");
// b.js
exports.stuff = function(text) {
    console.log(text);
}

... wird zu ... 

// bundle.js
({
"./a.js": function(module, exports, require) {
    var b = require("./b.js");
    b.stuff("It works");
},
"./b.js": function(module, exports, require) {
    exports.stuff = function(text) {
    console.log(text)
}})

Limitierungen

  • Keine C-Libs
  • Keine Betriebssystem-spezifischen Module (z.B. fs)
  • Kein dynamisches require() 
    var modulePath = "path/to/module.js";

require(modulePath);
